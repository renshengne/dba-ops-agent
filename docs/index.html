FOUR CAPABILITIES

四块能力，把资深 DBA 的肌肉记忆装进 Agent。

从被动救火到主动预判，覆盖巡检、诊断、性能、恢复全链路，全部只读——不碰你的数据。

01 / PATROL

智能巡检

配置巡检模板，cron 定时采集连接数、慢 SQL、锁、容量、复制延迟，自动比阈值生成正常/警告/严重三态报告。

  • MySQL · GaussDB/PG · Oracle 三方言
  • 严重项秒推飞书/钉钉告警卡片
  • 报告 Markdown + JSON 双产物
02 / DIAGNOSE

异常诊断

告警或提问触发后，自动串联多指标定位根因，输出"现象→证据→根因→建议"完整链路，不再在视图间反复横跳。

  • 锁等待链自动溯源到阻塞源会话
  • 连接数告警区分泄漏/慢SQL堆积/真实并发
  • 等待事件按 CPU/IO/锁/网络 归类
03 / PERF

性能诊断

慢 SQL 拉起执行计划、表统计、索引、关联等待事件，LLM 结合上下文给出瓶颈定位与索引/改写建议，标注待人工验证。

  • TOP SQL 热点锁定
  • 等待事件 TOP N 解读
  • 索引建议含写性能影响评估
04 / RECOVERY

快速恢复

诊断结论直接产出可执行脚本 + 影响评估 + 回滚预案，覆盖 kill 会话、主备切换、参数回滚、容量处置——只给手册，绝不代你动手。

  • 主备切换含前后检查清单
  • 参数回滚附验证指标
  • 每个动作 requires_human=true
AGENT PIPELINE

一条流水线，从意图到带预案的报告。

Agent 编排核心把采集、规则诊断、LLM 综合与恢复建议串成可流式回放的事件流。

STAGE 01
PLAN
解析意图，绑定实例与巡检模板，规划采集指标集。
STAGE 02
COLLECT
并发线程池跑只读采集，单指标失败不中断整体。
STAGE 03
DIAGNOSE
规则引擎阈值比对 + 根因推断 + 等待事件分类。
STAGE 04
ADVISE
LLM 综合根因报告，恢复建议器产出带预案的处置手册。
SAFETY BOUNDARY

它很能干，但有一条不敢越的红线。

Agent 只持只读账号、只发只读 SQL、只产出手不碰数据库——把"动"的权力永远留给 DBA。

READ-ONLY BY DESIGN

启动即校验账号权限：发现 INSERT/UPDATE/DROP 等写权限，直接拒绝启动（除非显式 allow_write_account）。每条采集 SQL 经 is_read_only_query 拦截，写语句根本进不了连接。恢复建议输出为纯文本脚本，代码中无任何 execute/commit/落库调用。
